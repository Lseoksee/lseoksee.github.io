    2024년 11월 07일11 min read

    의류 사이즈 정보 및 수선 개선

    수정할 API

    • {GET} 제품 상세 조회: http://fitpitback.kro.kr:8080/api/item-info/{itemKey}
    • {GET} 주문 조회: http://fitpitback.kro.kr:8080/api/order/get_order/{userEmail}
    • {GET} 장바구니 목록 조회: http://fitpitback.kro.kr:8080/api/cart/get-store/{userEmail}
    • {GET} 수선내역 조회: http://fitpitback.kro.kr:8080/api/pit/get/{cartKey}
    • {POST} 장바구니 저장: http://fitpitback.kro.kr:8080/api/cart/store
    • {POST} 주문내역 등록: http://fitpitback.kro.kr:8080/api/order/post_order

    생성할 API

    • {POST} 수선 가격 갱신 API

    item 테이블 수정

    • 수선 가격을 나타내는 fitprice 컬럼 하나 추가해 주세요
    • 일단은 모든 수선 가격을 2000 으로 값을 추가해 주세요
    • 이게 수선 기본 비용 입니다.

    itemTopInfo & itemBottomInfo 테이블 수정

    • 의류 각 파트를 이렇게 구성 해야 합니다 보니까 itemTopInfo 에 들어 가는 값이 5개 더군요
      • 상의
        • 총장, 어깨너비, 가슴단면, 소매길이
      • 하의
        • 총장, 밑위, 허리단면, 엉덩이 단면, 허벅지단면, 밑면 단면

    {GET} 제품 상세 조회

    • 위에서 item 테이블에 fitprice 컬럼 추가했잖아요.
    • 그거 그대로 반환하는 값을 하나 추가해 주세요

    응답

    {GET} 주문 조회

    • 수선을 한 경우 수선된 사이즈가 반환되야 합니다
    • 또한 주문한 날짜를 반환 해 주세요.
    • 참고로 pitPrice 값은 현재 상태 그대로 orderList 테이블 값 리턴하면 됩니다
      • displayPitPrice를 pitPrice로 변경합니다.
    • pitStatus 에 경우 지금처럼 “수선 있음” 없음으로 텍스트 형태로 반환하지 않고 true, false 로 boolean 형으로 반환 해 주세요

    응답

    {GET} 장바구니 목록 조회 & {GET} 수선내역 조회

    • 두 api 를 통합 합시다
    • 위 주문 조회 api 와 동일하게 pitStatus 키값을 추가합니다

    응답

    {POST} 장바구니 저장

    • 수선 여부 및 수선된 사이즈 pitTopInfo, pitBottomInfo 을 요청 body에 담아서 수선에 대한 처리를 할 수 있게 합니다.
      • 서버에서는 pitTopInfo 이 null 인지 pitBottomInfo이 null 인지를 판단하여 상의를 수선 인지, 하의를 수선 인지 판단 할 수 있습니다
    • pit 값은 없에 주세요

    요청 Body

    {POST} 주문내역 등록

    • 기존에 수선 여부로 사용한던 pit 키값을 pitStatus 로 바꾸고 boolean 타입으로 처리 해주세요
    • 요청에 쓰이는 pitPrice는 없에주세요
    • 위에서 말했듯이 item 테이블에 fitprice 컬럼을 추가 했잖아 해당 api 로 요청이 들어오면 itemKey 값에 맞는 수선 가격을 item 테이블에 fitprice 컬럼을 참조하여 orderList 테이블에 pitPrice 값으로 추가하면 됩니다.
    • 수선 여부 및 수선된 사이즈 pitTopInfo, pitBottomInfo 을 요청 body에 담아서 수선에 대한 처리를 할 수 있게 합니다.
      • 서버에서는 pitTopInfo 이 null 인지 pitBottomInfo이 null 인지를 판단하여 상의를 수선하는지 하의를 수선하는지 판단 할 수 있습니다
    • 또한 제품에 대한 부분은 [] 리스트 형태로 요청 합니다. 장바구니에 여러개 항목 담아서 구매할때 처리가 필요해서요

    요청 Body

    {POST} 수선 가격 갱신 API

    • 웹에서 경매를 마쳐서 최종적으로 수선 가격이 정해지면 orderList 테이블의 pitPrice 값이 업데이트 되야 합니다.

    요청 Body

    응답

    • 정상적으로 업데이트 되었다는 메시지

    장바구니 & 주문내역 수선 DB 구성에 대한 팁

    • 주문내역 orderList 테이블과, 장바구니 cart 테이블에 수선된 부분을 저장하는데는 좀 무리가 있기 때문에
    • 지금 PitTop, PitBottom 이 수선용 DB 인거 같은데
    • 이렇게 되면 주문 내역에 대한 것도 처리하려면 테이블만 4개를 생성해야 하니까
    • 이렇게 구성해 보면 어떨까 라고생각되서 적어봅니다

    장바구니 수선용 DB

    • 만약 수선을 선택하지 않은 경우에는 해당 테이블에 값이 존재하지 않겠죠
