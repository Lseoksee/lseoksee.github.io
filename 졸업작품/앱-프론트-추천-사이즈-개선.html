    앱 프론트 추천 사이즈 개선

    앱 프론트 추천 사이즈 개선

    2024년 11월 05일17 min read

    공통 확인 사항

    • itemTopInfo 값이 null 이냐 itemBottomInfo 값이 null 이냐에 따라서 상의 수선 하는지, 하의 수선 하는지 구분 가능합니다.

    작업용 계정

    채형 정보와, 장바구니, 결제 목록이 나와있는 계정 입니다. 해당 계정 사용하여 작업해 주세요

    • 이메일: da864268@naver.com
    • 비번: 1234

    의류 실측 구분

    • 상의: 4개
      • 총장 (itemHeight), 어깨너비 (itemShoulder), 가슴단면 (itemChest),
        소매길이 (itemSleeve)
    • 하의: 6개
      • 총장 (itemHeight), 밑위 (frontrise), 허리단면 (itemWaists),
        엉덩이 단면 (itemhipWidth), 허벅지 단면 (itemThighs),
        밑면 단면 (itemHemWidth)

    백엔드 수정된 API

    • {GET} 제품 상세 조회: http://fitpitback.kro.kr:8080/api/item-info/{itemKey}
    • {GET} 주문 조회: http://fitpitback.kro.kr:8080/api/order/get_order/{userEmail}
    • {GET} 장바구니 목록 조회: http://fitpitback.kro.kr:8080/api/cart/get-store/{userEmail}
    • {POST} 장바구니 저장: http://fitpitback.kro.kr:8080/api/cart/store
    • {POST} 주문내역 등록: http://fitpitback.kro.kr:8080/api/order/post_order

    제품 상세 페이지 (채빈)

    제품 사이즈 API 연동

    • 제품 상세 조회 API 사용: {GET} http://fitpitback.kro.kr:8080/api/item-info/{itemKey}
      • API에 대한 추가 설명은 백엔드 API 문서를 참고해 주세요.
    • 반환되는 사이즈에 따라 선택할 수 있는 사이즈(S,M,L)가 바꿔야 되며 지금은 L사이즈 하나만 표시되는데 모든 사이즈 전부 테이블에 표시되야 합니다.

    API 응답

    • itemTopInfo 값에 사이즈 정보가 배열 형태로 나오고 itemBottomInfo 값은 null로 나옵니다 이걸 이용해 해당 의류가 상의 인지, 하의 인지 구분 할 수 있죠

    추천 사이즈 정의

    • 체형 분석 결과 API 사용: http://fitpitback.kro.kr:8080/api/userbodyinfo/{유저이메일}

      • API에 대한 추가 설명은 백엔드 API 문서를 참고해 주세요.

    추천 사이즈 알고리즘

    • 진짜로 어떤 계산에 해서 정할껀 아니고 그냥 원래 사이즈를 채형 사이즈로 값만 대체 할껍니다
    상의
    • 총장 = 분석된 bodySize
    • 어깨너비 = 분석된 shoulderSize
    • 가슴단면 = 이건 그냥 원래 그대로
    • 소매길이 = 분석된 armSize
    하의
    • 총장 = 분석된 legSize
    • 나머지는 원래 값 그대로 씁니다

    페이지에서 수선하기 버튼을 체크 한 경우

    • 맨 하단에 제품 조회 api 서 나온 수선 가격 (fitprice) 가 하단에 표시되야 합니다

    사이즈 선택 버튼

    • 분석된 채형 사이즈 보다 한 단계 더 큰 사이즈로 선택되어 바꾸지 못하게 고정해야 합니다
      • 무슨 소리냐면 예를들어 총장 50, 어깨 너비 40, 소매길이 30 으로 채형이 측정 되었다고 가정해 봅시다
      • M 사이즈의 실측값이 총장 55, 어깨너비 35, 소매길이 30 으로 나온 경우
      • 총장과, 소매 길이는 분석된 채형 보다 더 크지만 어깨너비가 분석된 채형 보다 더 작으므로, 한 치수 더 큰 L사이즈로 선택이 되야겠죠
      • 이때 L사이즈가 조건을 충족하는 경우 해당 사이즈로 고정이 되야 됩니다.
      • 하지만 L사이즈도 만족을 못하면서 L사이즈가 가장 큰 사이즈 인 경우 그냥 L사이즈로 고정하면 됩니다. (어쩔 수 없으니까요)
      • 왜 이렇게 하냐면 수선을 하려면 결국은 의류를 자르는 행위인데 더 작은 사이즈에서 나에게 맞게 수선 한다는건 불가능 하잖아요 그리고 너무 큰 사이즈에서 수선 할 필요도 없고요
    • 다시 수선하기 버튼을 해제 한다면 사이즈를 변경할 수 있게 하면 됩니다.

    원래 사이즈, 추천사이즈 비교 표

    • 위에서 결정된 사이즈(S,M,L…)의 원래 실측 크기와
    • 채형 측정으로 결정된 추천 사이즈를 보여 주시면 됩니다.

    사이즈 조정 버튼

    • 현재는 상의만 되있고 하의는 없어서, 상의, 하의에 대해서 처리가 되여 합니다
    • 조정을 하면 그거에 맞게 추천 사이즈 에서 +- 가 되어서, 장바구니에 저장 하거나, 구매 하게 설계가 들어가야 합니다.

    장바구니 담기 API 연동 수정

    • 장바구니 저장 API: {POST} http://fitpitback.kro.kr:8080/api/cart/store
      • API에 대한 추가 설명은 백엔드 API 문서를 참고해 주세요.
    • handleAddToCart() 쪽 변경 하시면 됩니다

    API 요청 Body

    • 수선을 선택 한 경우 pitStatus 값을 true 아니면 false 로 값을 주면되고
    • 수선이 아닌경우 pitTopInfo, pitBottomInfo 두 값을 null 로 보내주시면 됩니다

    바로 구매

    • 값을 아래처럼 만들어서 navigate param 으로 array로 전달 하시면 됩니다
    • handlePurchase() 수정 하시면 됩니다

    장바구니 페이지 (주희)

    장바구니 페이지 API 연동 수정

    • 장바구니 목록 조회 API: {GET} http://fitpitback.kro.kr:8080/api/cart/get-store/{userEmail}
      • API에 대한 추가 설명은 백엔드 API 문서를 참고해 주세요.

    API 응답

    • pitTopInfo 이 null 이냐 pitBottomInfo 이 null 이냐에 따라 해당 의류가 상의인지 하의인지 구분할 수 있습니다.

    수선 버전에 대한 표시 및 디자인

    • 위 수선 여부(pitStatus) 값에 따라 수선 버전용으로 바꿔서 보여줘야 합니다
    • 당연하지만 상의, 하의에 대해서도 처리를 해야죠

    주문하기 버튼 이벤트 수정

    • 지금은 주문하기를 누르면 그냥 Order 페이지로 넘어가게만 설계되어 있는데
    • 장바구니 페이지 API 에서 넘겨받은 상품 정보들을 아래처럼 navigate param 으로 넘겨서 Order 페이지로 넘어가야 합니다
    • 참고로 장바구니 목록 조회 API 응답 데이터랑 살짝 다릅니다. 아래 나온 것처럼 값을 맞춰주세요
    • 왜 이렇게 하냐면 제품 상세에서 바로 구매 할때랑 일관성을 맞추기 위함입니다.

    장바구니 삭제 구현

    • X 버튼같은거 하나 추가해서 장바구니 삭제를 구현합니다
    • 장바구니 삭제: {DELETE} http://fitpitback.kro.kr:8080/api/cart/delete
      • API에 대한 추가 설명은 백엔드 API 문서를 참고해 주세요. (참고로 제품 상세 페이지 부분에 설명이 있습니다)

    주문 결제 페이지 (주희)

    결제 API 연동 수정

    • 주문내역 등록 API: {POST} http://fitpitback.kro.kr:8080/api/order/post_order
      • API에 대한 추가 설명은 백엔드 API 문서를 참고해 주세요.
    • postOrder() 부분 수정하시면 됩니다
    • 아직 우진님이 여기 백엔드를 수정하지 않아 나중에

    API 요청 Body

    • 수선을 선택 한 경우 pitStatus 값을 true 아니면 false 로 값을 주면되고
    • 수선이 아닌경우 pitTopInfo, pitBottomInfo 두 값을 null 로 보내주시면 됩니다

    장바구니 목록 불러오기 수정

    • 위 장바구니 페이지와, 제품상세/바로구매 쪽에서 둘다 navigate param 으로 처리 되어 더 이상 장바구니 api를 호출 할 필요가 없습니다

    • fetchCartItems()이쪽 수정하시면 됩니다

    • 아마 해당 함수 지우고 최상단에 이렇게만 바꾸고, 장바구니 에서 설정한 navigate param 값을 참고하여 몇가지만 수정하시면 됩니다.

    수선 버전에 대한 표시 및 디자인

    • 위 장바구니 처럼 수선 버전에 대한 디자인이 필요합니다.
    • figma 보니까 그냥 수선해서 구매하기 체크박스 하나만 있던데 장바구니 처럼 수선한 사이즈 볼 수 있게 해야 합니다.
    • 당연하지만 상의, 하의에 대해서도 처리를 해야죠
