    자료구조 기말

    자료구조 기말

    2025년 2월 02일9 min read

    이중 연결리스트

    • 삽입시

    스택

    • 후입후출(마지막에 온게 먼저)

    구성요소

    • isEmpty(): 공백인지 아닌지 리턴
    • isFull(): 배열에 끝인지 리턴
    • push(): 삽입
    • pop() 삭제(맨 마지막에 온놈)
    • printStack(): 스택 순회

    배열스택 과정

    1. top 을 -1 로 설정
    2. push() 하면 isFull() 확인하고 ++top 하고 아이템삽입
    3. pop() 하면 isEmpty() 확인하고 top 위치꺼 리턴해줌

    연결리스트 스택 과정

    1. push() 하면 이전에 head를 새로운 노드에 링크하고 head를 넘김
    2. pop() 하면 isEmpty() 확인하고 head꺼 리턴하고 head를 다음 링크로 넘기고 free()

    연결리스트 스택으로 괄호판단

    • 여는괄호면 push() 닫는괄호면 pop()

    큐

    • 선입선출(먼저온게 먼저나감)

    구성요소

    • front: 삭제된 지점을 가르키는 변수
    • rear: 삽입된 지점을 가르키는 변수
    • isEmpty(): front=rear 면
    • isFull(): rear가 마지막 인덱스면
    • enQueue(): 큐 삽입
    • deQueue(): 큐 삭제

    배열 큐 과정

    1. front, rear 를 -1로 초기화
    2. enQueue() 하면 isFull() 검사하고 ++rear 하고 arr[rear] 에 삽입
    3. deQueue() 하면 isEmpty() 검사하고 ++front 하고 삭제된거 리턴

    원형 큐 과정

    1. front, rear 를 0으로 초기화 (0으로 초기화 하는 이유는 순환구조 특성상 하나를 무조건 비워나야함)
    2. isFull() 함수는 (rear+1)%size 한게 front랑 같으면
    3. enQueue(), deQueue() 할 시 무조건 size 나머지 연산자 쓸것
    4. printQueue()에 경우 rear, front 값을 +1 해서 불러옴 전부 위에처럼 나머지 연산자 쓸것 (오버플로우 나기때문), front 부터 rear 까지 출력하면 됨

    연결리스트 큐 과정

    1. head 를 front, rear 이렇게 생성
    2. isEmpty() 함수에 경우 front==null && rear == null
    3. enQueue()시 isEmpty()면 front 와 rear를 새로운노드에 위치시킴 아니면 rear→link 에 삽입하고 rear를 새로운노드로 위치시킴
    4. deQueue()시 isEmpty() 판단하고 front를 다음노드로 가르키게 하고 이전노드 삭제 후 해당노드 데이터 리턴 이때, 다음노드가 null 인경우 front 도 null로 만듬

    데크

    • front, rear 에서 삽입, 삭제가 동시에 되는 구조, 이중 연결리스트 사용

    구성요소

    isEmpty(): front==null && rear == null 이면 insertFront(): 앞쪽에 삽입 deleteFront(): 앞쪽에 삭제 insertRear(): 뒤쪽에 삽입 deleteRear(): 뒤쪽에 삭제

    과정

    1. Front 요소는 무조건 head가 앞에 존재해야 하고 Rear 요소는 무조건 뒤에 존재해야함
    2. 나머지는 연결큐와 동일
    3. deleteFront() 시 Llink를 NULL 로 만들어야하고 deleteRear() 시 Rlink를 NULL 로 만들어야함

    트리

    용어

    • 노드: 트리원소
    • 간선: 노드를 연결하는 선
    • 형제노드: 같은 부모 노드
    • 서브트리: 부모노드와 연결된 간선을 끊었을때 생기는 노드 즉 자식이 있으면 서브트리
    • 차수: 노드에 연결된 자식 노드 수

    이진트리

    • 트리에 모든 노드이 차수를 2이하로 제한
    • 최대 노드개수 2^(h+1)-1
    • 포화 이진트리: 모든 노드가 2개씩 가득 차 있는경우
    • 완전 이진트리: 왼쪽부터 자식을 채워 넣었고 포화상태가 아닌경우

    배열 이진트리

    • 위 사진 순서대로 저장
    • 노드 n에 부모: n/2
    • 노드 n에 왼쪽 자식: 2*n
    • 노드 n에 오른쪽 자식: 2*n+1

    연결리스트 이진트리

    • 왼쪽자식 주소, 오른쪽 자식 주소 각각 구현

    이진트리 순회

    • 전위 순회: 나→ 완쪽 → 오른쪽 A →B →D→H→I→E→C→F→G
    • 중위순회: 오른쪽→나→왼쪽 H→D→I→B→E→A→F→C→G
    • 후위순회: 왼쪽→오른쪽→나 H→I→D→E→B→F→G→C→A

    이진 탐색 트리

    • 왼쪽은 루트보다 낮은 수
    • 오른쪽은 루트보다 큰 수
    • 서브트리도 이진 탐색 트리

    찾으려는 인덱스가 루트보다 작으면 왼쪽, 높으면 오른쪽

    • 삽입, 인덱싱은 모두 제귀함수 쓰면 됨
    • 삭제는 3가지로 나뉨
      1. 자식이 없는경우 그냥 삭제하면 됨
      2. 단일자식만 있는경우 자기자리를 자식한테 넘기고 삭제
      3. 두 자식을 가진경우 오른쪽 자식 중 가장 작은 자식을 후계자로 삼던가 왼쪽 자식 중 가장 큰 자식을 후계자로 삼던가

    힙

    • 완전 이진트리 상황에서 가장 큰 값, 또는 가장 작은 값 찾기위해
    • 루트가 제일 큰 값이면 최대힙, 작은값이면 최소힙
    • 즉 모든 서브 트리에 자식요소는 부모보다 작거나 크면 됨
    • 삽입은 끝 쪽에 임시로 놔두고 부모랑 대소비교 해서 삽입될 위치 파악해서 삽입 힙 삽입.pdf
    • 삭제는 무조건 루트노드 삭제, 맨 마지막 원소를 루트에 넣고 자손들이랑 대소비교 해서 교환 힙 삭제.pdf

    정렬

    • 버블정렬
    • 선택정렬 0번부터 인덱스를 늘려가며 작은 수 찾아서 위치 바꾸는 방식
    • 삽입정렬 정렬한 배열에서 정렬될 원소가 들어갈 위치를 찾아 삽입하는 구조

    검색

    순차검색, 선형검색

    • 배열이나 연결리스트에서 원하는 항목을 찾는 방법
    • 정렬된 자료에 경우 찾는 값이 배열에 있는 항목보다 크면 바로 종료할수있기에 좀더 빠름
