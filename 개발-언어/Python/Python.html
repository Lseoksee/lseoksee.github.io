    Magic Method

    • __ 로 시작해서 끝나는 기본 클래스 정의 메소드
    • 이것 이외의 인덱스 관련함수 목록

    __getitem__(index)

    • 객체의 [] 연산자를 사용하여 조회 할 때 작동하는 메소드
    • numpy의 [3,4] 이런 슬라이싱 방식이 이걸로 구현되어있다
    • list[10] == list.__getitem__(10)

    __setitem__(index, key)

    • 객체의 [] 연산자를 사용해서 변수를 지정할 때
    • list[10] = 1 == list.__setitem__(10, 1)

    __str__()

    • 객체의 직접 print 를 줄시 나오는 출력 구문 커스텀

    __call__()

    • 객체를 함수처럼 호출할 수 있게 해주는 메소드

    enumerate

    • 리스트 행목들로 for 돌릴시 인덱스도 같이 얻는 객체

    typing

    Literal

    • 타입을 “이거” “저거”로 만 명시 하는
    • 특정 Literal 따라 타입이 동적으로 변해야 하는경우는 명시 불가

    딕셔너리 안에 딕셔너리 typing

    • 이때는 dict 의 생성자를 사용해 typing 가능하다
    • 그러면 딕셔너리["키1"]["항목1"] 까지 typing 된다

    Sequence

    • 이건 list 와 tuple 같은 열거형 자료형을 의미

    GIL (Global Interpreter Lock) 문제

    Python에 제일 큰 문제는 1코어 1쓰레드로 밖에 동작하지 못한다

    • 만약 threading API로 여러 쓰레드로 작업한다고 해도 GIL 땜에 사실상 동시에 실행된다는 전제는 아니다
    • 다만 완전하게 의미는 없는게 아닌게 I/O 작업같이 Windows API 같은 OS API 의 의존하거나, 라이브러리 자체가 C, C++로 구현되있는 경우 GIL이 관여하지 않으므로 멀티쓰레딩 사용이 가능하다
    • 하지만 순수 Python 코드로 되어있는 CPU 연산작업에는 별로 큰 의미가 없어진다
      • 이떄는 사실상 비동기와 큰 차이가 없어지는 것이다. 단지 구조적인 차이와, 비동기 관리를 언어 수준에서 하냐, OS 수준에서 하냐 정도
    • Python 3.13 이후에는 이게 선택으로 바뀐다는데 아직까지 제대로된 정보를 잘 모르겠다

    multiprocessing

    파이썬에서 멀티프로세싱을 수행하는 객체

    • Python 에서는 이런 GIL 문제를 우회하고자 멀티프로세싱을 많이 쓰는거 같다
    • 다만 Frozen Start 문제 때문에 별로 오래 안걸리는 작업에는 멀티쓰레딩이 몇배는더 빠르다

    설명

    • f() 함수를 각각 1, 2, 3, 4, 5의 인자를 전달받아 연산한다는 개념
    • 만약 Pool 을 3으로 변경하면 3개의 프로세스를 1, 2, 3에 할당하여 연산 시키고, 가장 빨리 끝난 순으로 나머지 4, 5를 연산

    멀티프로세싱의 특징

    • 현재 프로그램을 실행하는 프로세스와 완전히 다른 프로세스를 생성하는것
    • 독립된 Memory-Map 과, 자원을 할당받음
    • 같은 Memory-Map을 쓰지 않으므로 각각의 프로세스 간 통신은 IPC로 이루어짐

    Asterisk

    Python 에서의 *

    Unpacking (*)

    일반 코드에서의 사용

    • 마치 JS 의 ... 연산자 처럼 리스트나, 튜플을 풀어주는 역할을한다
    • 이렇게 여러개의 인자를 전달해야 하는경우 list나 tuple을 만들어서 전달도 가능해진다
    • 딕셔너리의 경우 키값이 Unpacking 된다

    positional argument (*)

    함수에서 사용

    • 함수 인자를 튜플 형태로 받는다
    • 아래 keyword arguements 와 다르게 키값이 필요하지 않아 그냥 무작위 값을 마음데로 넣을 수 있다
    • 모호성 문제로 positional argument을 두번 쓰는건 허용하지 않는다

    keyword arguements (**)

    함수에서 사용

    • 함수 인자를 동적 딕셔너리로 받는다
    • 함수 호출 시 kwargs 에는 반드시 key-value 형태로 전달해야 함
    • 모호성 문제로 keyword arguements을 두번 쓰는건 허용하지 않는다
